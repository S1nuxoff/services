# Пакети розробки сайтів та Telegram-ботів

---

# 1. Landing Basic

**Вартість: від $100**

### Опис

Готовий акуратний лендінг на 1–2 сторінки з анімаціями, адаптивом та формою для заявок. Працює на власному домені, має HTTPS і базові SEO-теги. Підходить для презентації послуг або запуску реклами.

### Склад робіт

- Розмітка та стилізація інтерфейсу за макетом Figma.
- 1–2 сторінки.
- Адаптивна верстка (mobile/tablet/desktop).
- Базові анімації (CSS/Framer Motion).
- Форма заявки (Email/Telegram API).
- Налаштування домену, SSL, хостингу.
- Мінімальні SEO-теги (title, description, open graph).

---

# 2. Landing PRO / Business Site

**Вартість: від $200–300**

### Опис

Повноцінний бізнес-сайт із каталогом, сторінками товарів, корзиною та онлайн-оплатами. Вже готовий до запуску реклами, збору заявок і продажів. Є аналітика та серверна логіка.

### Склад робіт

- Розробка 2–6 сторінок за Figma.
- Каталог/листинг товарів.
- Сторінка товару з динамічними даними.
- Локальне збереження корзини (localStorage/state).
- Інтеграція оплат (Monobank API, LiqPay API, Stripe Checkout/Elements).
- Прості серверні обробники (FastAPI/Node.js).
- Підключення аналітики (GA4 / Meta Pixel).

---

# 3. Service Platform / Mini-SaaS

**Вартість: від $350–600**

### Опис

Міні-сервіс або SaaS-платформа з особистими кабінетами, адмінкою, ролями, базою даних і повноцінним API. Використовується для автоматизації бізнес-процесів, внутрішніх систем або запуску MVP стартапу.

### Склад робіт

- 5–10 сторінок або модулів інтерфейсу.
- Авторизація та реєстрація (JWT, refresh, role-based access).
- Особистий кабінет (CRUD-операції по даних).
- Адмін-панель (користувачі/дані/контент).
- База даних: PostgreSQL/MySQL.
- Бекенд: FastAPI або Node.js (REST API).
- Інтеграції (зовнішні API, webhook-и).
- Підтримка WebSockets (оновлення в реальному часі).
- Деплой на VPS (Docker + Nginx).

# Пакети Telegram-ботів

## 1. Basic Bot

**Вартість: від $40–70**

### Опис

Бот із простими сценаріями, кнопками та збором заявок. Підходить для консультацій, прийому контактів, опитувань і початкових автоматизацій.

### Функціонал

- Меню (reply/inline keyboards).
- Прості сценарії (aiogram FSM).
- Збір форм/заявок.
- Опціонально — логування в БД.

---

## 2. Bot з адмін-панеллю

**Вартість: від $80–150**

### Опис

Бот із системою керування: можна переглядати користувачів, заявки, змінювати тексти та контент. Працює як компактна CRM.

### Функціонал

- Користувацьке меню.
- Адмін-панель (бот або веб-інтерфейс).
- Перегляд користувачів, заявок.
- Редагування текстів, блоків, контенту.
- Збереження даних у PostgreSQL/SQLite.

---

## 3. Бот з оплатами

**Вартість: від $150–250**

### Опис

Бот із підтримкою онлайн-оплат: продажі, підписки, внутрішній баланс. Ідеально підходить для курсів, товарів, платного доступу.

### Функціонал

- Оплати: Monobank/LiqPay/Stripe/Telegram Stars API.
- Баланс/тарифи.
- Обробка транзакцій (webhook).
- Нотіфікації в адмін-канал.
- Простий бекенд (FastAPI/Node.js).

---

## 4. Складний бот

**Вартість: від $250–500+**

### Опис

Багатофункціональний бот з великою логікою: кабінети, реферальна система, інтеграції з API, ролі, статистика. Підходить для стартапів і сервісів.

### Функціонал

- Розгалужена FSM-логіка.
- Реферальна модель.
- Кабінет користувача.
- API-інтеграції (REST).
- Адмін-панель з ролями.
- Зберігання станів у БД.

# Стек технологій

## Frontend

- **React**, **Next.js**
- **TypeScript**
- **SCSS**, **TailwindCSS**
- **Framer Motion**
- REST/WebSocket
- Оптимізація (code splitting, memoization, lazy loading)

## Backend

- **FastAPI** (Python)
- **Node.js**: Express, NestJS
- Авторизація: JWT/OAuth2
- REST API, background tasks
- Інтеграції з зовнішніми API

## Бази даних

- PostgreSQL, MySQL, SQLite
- ORM: SQLAlchemy, Prisma, TypeORM
- Проєктування схем, оптимізація запитів

## DevOps

- Docker, Docker Compose
- Ubuntu VPS
- Nginx (reverse proxy + SSL)
- Cloudflare (DNS/HTTPS/Proxy)
- CI/CD (GitHub Actions)

## Telegram Bots

- Aiogram 2/3
- Оплати (Monobank/LiqPay/Stripe/Stars)
- FSM, callback-логіка
- Адмін-панелі (бот/веб)
- Webhook-архітектура

## Парсинг та збір даних

- HTTP-клієнти: `requests`, `httpx`, `cloudscraper`
- HTML-парсинг: `BeautifulSoup4`, `lxml`, регулярні вирази
- Робота зі складними структурами (каталоги, таблиці, списки)
- Побудова стабільних парсерів зі збереженням у БД
- Обхід первинних захистів (headers, cookies, сесії)
- Планувальники (cron/скрипти)
